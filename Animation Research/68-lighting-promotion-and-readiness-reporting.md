# 68 — Lighting Promotion and Readiness Reporting

Date: 2026-03-31

## Purpose
Define the reports that should exist before lighting-aware assets can be promoted or considered ready.

## Required report types

### 1. Lighting material audit
Confirms that required material families are tagged and protected.

### 2. Lighting profile library check
Confirms that the expected reusable profile library exists.

### 3. Lighting readiness report
Scores whether a character/location pairing is sufficiently lit for:
- dialogue
- performance
- production

### 4. Lighting routing case report
Confirms that routing fixtures include lighting complexity, not just motion complexity.

## Promotion policy
Do not promote a lighting-critical reference asset unless the reports indicate:
- profile compatibility exists
- material responses are present
- readability protections are in place
- representative relight review passed

## Handoff expectation
The handoff bundle outputs directory should contain these reports so engineers start from audited inputs.
