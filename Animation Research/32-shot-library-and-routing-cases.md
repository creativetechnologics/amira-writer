# 32 — Shot Library and Routing Cases

Date: 2026-03-31

## Purpose
Create a reusable library of example shot types so routing choices are not reinvented each time.

## Example shot families

### Dialogue close-up
- usually internal
- high revision sensitivity
- low environment complexity

### Singing medium close-up
- internal if mouth package is ready
- hybrid if lighting/FX layer is unusually hard

### Walk-and-talk medium shot
- internal if locomotion package exists
- hybrid if environment/parallax pressure is moderate

### Crowd spectacle wide shot
- often ai-video-fallback
- package coverage for every figure is rarely worth it

### FX-heavy transition shot
- hybrid or ai-video-fallback depending on retake needs

## Routing dimensions
- package readiness
- character count
- locomotion complexity
- prop complexity
- camera complexity
- environment FX
- revision sensitivity

## Suggested library fields
- shotId
- label
- category
- defaultRouting
- notes
- exampleParameters

## Production rule
The shot library should make routing feel like an informed default, not a fresh debate every time.
