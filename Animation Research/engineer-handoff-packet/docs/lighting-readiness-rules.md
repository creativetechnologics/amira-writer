# 62 — Lighting Readiness and Promotion Rules

Date: 2026-03-31

## Purpose
Define how lighting support changes package readiness and asset promotion.

## Lighting readiness levels

### Draft lighting
- ad hoc profile tests only
- missing response tags allowed
- not suitable for real shot planning

### Dialogue-ready lighting
- face and mouth readability protected
- active costume materials tagged
- at least day / sunset / night profile compatibility reviewed

### Performance-ready lighting
- required costume packs have response metadata
- practical-light interior profile tested
- background zone grading tested for core locations

### Production-ready lighting
- hero character and key locations pass the standard lighting profile set
- lighting QA loop exists
- relight failures are categorized as edit / regenerate decisions
- handoff packet contains lighting fixtures and validation outputs

## Promotion policy with lighting
Reference-driving assets should not be promoted if:
- they become unreadable under common day/sunset/night profiles
- line art collapses under expected shadow bias
- face or mouth visibility fails in ordinary dialogue conditions

Promote confidently when:
- the asset survives the representative relight set
- identity remains stable under warm/cool shifts
- costume colors remain distinct after palette clamp

## Minimum lighting profile set for hero characters
- daylight_soft
- sunset_warm
- moonlight_blue
- one dominant interior practical profile

## Practical takeaway
Lighting compatibility is now part of package quality, not a separate cosmetic layer.
